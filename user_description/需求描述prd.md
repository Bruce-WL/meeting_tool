# 需求目标
将上传的会议录音文件，整理成一份会议纪要

注释：
- PRD 用于定义页面结构与数据字段优先级，细节以实现代码为准。

# 功能模块
## 前端：面向用户，支持用户上传录音文件，查看录音纪要
  - 网页标题：AI会议纪要
  - 功能描述：上传录音后，自动生成AI纪要
  - 录音文件上传区域

## 后端：调用大模型将录音转文本，然后按要求的结构和样式，整理录音文件
  - 预期输出的会议纪要格式，以前端结果页与 `src/types/meeting.ts` 为准
  - 结果页分为以下几个部分
    - 标题
    - 总结
      - 一张结构图，通过卡片整理会议纪要的所有要点，结构请参考pdf文档
      - 分点总结
    - 待办
    - 智能章节
      - 根据视频内容分段，每一段前展示时间，点击时间可以跳转到用户上传视频对应的位置
    - 关键决策
    - 金句时刻

# 预期输出的格式说明
- 结构与字段以 `src/types/meeting.ts` 与前端结果页为准
- 视觉表现以页面当前实现为准


# 技术栈要求：

1. 使用 React + Tailwind CSS。
- 集成 Lucide-react 图标库。
- 禁止使用传统的渲染图库（如 Mermaid），请直接用 Tailwind 代码构建高保真卡片。

2. 页面布局结构（参考 image_fe0f9d.png）：
- 标题区：醒目的大标题，带日期和参会人信息。
- 总结区：包含核心要点卡片组。
- 待办事项：带复选框的清单，支持完成状态样式。
- 智能章节：左侧带时间戳（如 00:15），点击可高亮，右侧为分段摘要。
- 关键决策 & 金句时刻：使用特殊的背景色块进行强调。

3. 视觉规范（参考 image_fe29c3.jpg）：
- 核心卡片：每行 3 列，白色背景，rounded-2xl 圆角，细微灰色边框。
- 卡片内部：顶部左侧为图标+标题，右侧为带有浅紫色背景的 Badge（胶囊标签）。
- 分割线：卡片中间使用 border-t border-dashed 虚线。
- 列表符号：使用实心圆点，并确保文字换行时对齐。    

4. 数据驱动要求：
- 请先定义一个 const meetingData 模拟 JSON 对象。
- 页面所有内容必须从该 JSON 对象中读取，以便我后续对接后端 AI 接口。
- 请先为我生成这个页面的完整静态原型代码。
